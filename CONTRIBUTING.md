# Contributing

Thanks for considering contributing to KryptoKE. Small guidelines to make contributions smooth:

1. Fork the repo and create a feature branch from `main`.
2. Keep changes small and focused. One feature/bug per PR.
3. Follow existing code style. Backend uses Ruff settings from `backend/pyproject.toml`.
4. Run tests and linters before opening a PR.
5. Write a clear PR description and include screenshots when UI changes are involved.

If you're unsure where to start, open an issue describing the change and we can discuss.
